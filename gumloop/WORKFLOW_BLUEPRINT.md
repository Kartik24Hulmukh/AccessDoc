# Gumloop OSS workflow

1. Inputs/run ID → connection preflight → safe ZIP hash/extraction → source approval.
2. Repository inventory → license/privacy/secret/dependency/workflow scans in parallel.
3. Branch/import → reproducible build → compile/unit/HTTP/browser/PDF/security/commerce-absence tests.
4. Bounded defect loop (maximum five evidence-backed code cycles).
5. Human staging gate → deploy exact digest → HTTPS/header/smoke/load/alert/rollback tests.
6. Independent security/accessibility review → mechanical readiness → human production gate.
7. Immutable release, checksum, SBOM/provenance/evidence bundle.
8. Consented practitioner intake → redacted feedback → human-reviewed issues → weekly ethical adoption report.

Score: source/reproducibility 12, automated quality 18, security/privacy 18, accessibility/claims 15, deployment/TLS 12, observability/rollback 10, OSS governance 10, evidence/approvals 5. Public production requires all hard gates and ≥95. Traction is a separate ledger.
