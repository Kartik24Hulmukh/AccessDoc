# Release gates

A public release requires named human approval for three independent tracks.

## Repository-verified
- Clean install and test run in hosted CI.
- Unit, HTTP, container smoke, SBOM, and high/critical vulnerability checks recorded.
- Placeholder, stale-file, secret-pattern, and public-copy checks pass.

## Target-verified
- Ten-session practitioner protocol completed.
- Report corrections and second-use intent recorded.
- At least one independent accessibility practitioner reviews generated output.

## Editorial and legal approved
- Project name receives a documented clearance review.
- Copyright owner, release owner, and security contact are named.
- Public copy is reviewed for scanner, certification, conformance, privacy, and retention claims.

Until all tracks pass, describe the project as an experimental beta and do not operate a public upload service.
