# Private operator compliance note

**Draft template - not legal advice - requires qualified counsel review before operative use.**

Before public operation, identify operator entity, target jurisdictions, data roles, lawful basis where applicable, retention/deletion, subprocessors, international transfers, security measures, incident duties, accessibility obligations for the service itself, consumer/marketing rules, open-source notices, trademark clearance and vulnerability-disclosure process.

A neutral project name, local installation option or disclaimer does not remove obligations. Duties attach to operator conduct, users, data, claims and jurisdictions. Keep this note private if it contains counsel advice or operational details; publish an accurate user-facing privacy notice separately.
