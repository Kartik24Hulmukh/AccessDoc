# Evidence-ranked roadmap

The project should improve the evidence handoff rather than become another scanner or compliance dashboard.

| Priority | Idea | Evidence gate before expansion | Kill condition |
|---|---|---|---|
| P0 | Practitioner workflow comparison | Five completed evaluations | Fewer than three sendable results |
| P0 | HTML companion task testing | At least one regular screen-reader user | Critical tasks remain blocked after two revisions |
| P1 | Catalog diff and mapping provenance | Practitioners report mapping-review friction | No observed use in five evaluations |
| P1 | Static unmapped-findings worksheet | Three real exports contain consequential unknowns | Reviewers prefer source tools |
| P1 | Offline receipt verification | Two recipients preserve or verify sidecars | No named recipient workflow |
| P1 | Sanitized compatibility corpus | Twenty permitted real-world samples | Adapter failures cannot be reproduced safely |
| P2 | Report-local finding cross-references | Review sessions show reference ambiguity | Reviewers never use IDs |
| P2 | Tagged-PDF research path | Specialist confirms PDF remains required | HTML is preferred and accepted |
| Reject | Scanner, certification, legal conclusions, guessed AI mappings | Outside current evidence | Do not build |
| Reject | Persistent client dashboard or hosted verification service | Requires repeated-use evidence and security model | Fewer than four second-use requests after ten evaluations |
